17 *
18 * Persisted filters are attributed to a filter group so that all filters
19 * of the same kind can be discovered from the database.
20 *
21 * Persisted filters also can provide a list of FilterSetting objects which
22 * represent persistable filter configuration parameters.
23 *
24 * Persisted filters are templatable. This means that a non-parameterized
25 * copy of the filter can be persisted as a template for actual filter
26 * instances. The end user can discover such templates and use them to
27 * configure personalized transformations.
28 *
29 * The filter template can provide default settings for all filter
30 * settings.
31 *
32 * Persistable filters can be accessed via the FilterDAO which acts as a
33 * filter registry.
34 *
35 * Filters can be organized hierarchically into filter networks or
36 * filter pipelines. The hierarchical relation is represented via parent-
37 * child relationships. See CompositeFilter for more details.

255    /**
256     * There are certain generic filters (e.g. CompositeFilters)
257     * that sometimes need to be persisted and sometimes are
258     * instantiated and used in code only.
259     *
260     * As we don't have multiple inheritance in PHP we'll have
261     * to use the more general filter type (PersistableFilter) as
262     * the base class of these "hybrid" filters.
263     *
264     * This means that we carry around some structure (e.g. filter
265     * groups) that do only make sense when a filter is actually
266     * being persisted and otherwise create unnecessary code.
267     *
268     * We provide this helper function to instantiate a temporary
269     * filter group on the fly with only an input and an output type
270     * which takes away at least some of the cruft.
271     *
272     * @param string $inputType

278        $temporaryGroup->setSymbolic(FILTER_GROUP_TEMPORARY_ONLY);
279        $temporaryGroup->setInputType($inputType);
280        $temporaryGroup->setOutputType($outputType);
281        return $temporaryGroup;
282    }
