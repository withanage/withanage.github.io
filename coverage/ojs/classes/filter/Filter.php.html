11 * @ingroup filter
12 *
13 * @brief Class that provides the basic template for a filter. Filters are
14 *  generic data processors that take in a well-specified data type
15 *  and return another well-specified data type.
16 *
17 *  Filters enable us to re-use data transformations between applications.
18 *  Generic filter implementations can sequence, (de-)multiplex or iterate
19 *  over other filters. Thereby filters can be nested and combined in many
20 *  different ways to form complex and easy-to-customize data processing
21 *  networks or pipelines.
22 *
23 *  NB: This also means that filters only make sense if they accept and
24 *  return standardized formats that are understood by other filters. Otherwise
25 *  the extra implementation effort for a filter won't result in improved code
26 *  re-use.
27 *
28 *  Objects from different applications (e.g. Papers and Articles) can first be
29 *  transformed by an application specific filter into a common format and then
30 *  be processed by application agnostic import/export filters or vice versa.
31 *  Filters can be used to pre-process data before it is indexed for search.
32 *  They also provide a framework to customize the processing applied in citation
33 *  parsing and lookup (i.e. which parsers and lookup sources should be applied).
34 *
35 *  Filters can be used stand-alone outside PKP applications.
36 *
37 *  The following is a complete list of all use-cases that have been identified
38 *  for filters:
39 *  1) Decode/Encode
40 *  * import/export: transform application objects (e.g. an Article object)
41 *    into structured (rich) data formats (e.g. XML, OpenURL KEV, CSV) or
42 *    vice versa.
43 *  * parse: transform unstructured clob/blob data (e.g. a Word Document)
44 *    into application objects (e.g. an Article plus Citation objects) or
45 *    into structured data formats (e.g. XML).
46 *  * render: transform application objects or structured clob/blob data into
47 *    an unstructured document (e.g. PDF, HTML, Word Document).
48 *
49 *  2) Normalize
50 *  * lookup: compare the data of a given entity (e.g. a bibliographic
51 *    reference) with data from other sources (e.g. Crossref) and use this
52 *    to normalize data or improve data quality.
53 *  * harvest: cleanse and normalize incoming meta-data
54 *
55 *  3) Map
56 *  * cross-walk: transform one meta-data format into another. Meta-data
57 *    can be represented as structured clob/blob data (e.g. XML) or as
58 *    application objects (i.e. a MetadataRecord instance).
59 *  * meta-data extraction: retrieve meta-data from OO entities
60 *    (e.g. an Article) into a standardized meta-data record (e.g. NLM
61 *    element-citation).
62 *  * meta-data injection: inject data from a standardized meta-data
63 *    record into application objects.
64 *
65 *  4) Convert documents
66 *  * binary converters: wrap binary document converters (e.g. antidoc) in
67 *    a well-defined and re-usable way.
68 *
69 *  5) Search
70 *  * indexing: pre-process data (extract, tokenize, remove stopwords,
71 *    stem) for indexing.
72 *  * finding: pre-process queries (parse, tokenize, remove stopwords,
73 *    stem) to access the index
74 */

381

383            $hasRuntimeSettings = false;
384            foreach ($runtimeSettings as $runtimeSetting => $defaultValue) {

386                    $$runtimeSetting = $this->getData($runtimeSetting);
387                    $hasRuntimeSettings = true;

389                    $$runtimeSetting = $defaultValue;
390                }
391            }
392
393            // If we found any runtime restrictions then construct a
394            // runtime environment from the settings.
395            if ($hasRuntimeSettings) {
396                $this->_runtimeEnvironment = new RuntimeEnvironment($phpVersionMin, $phpVersionMax, $phpExtensions, $externalPrograms);
397            } else {
398                // Set null so that we don't try to construct
399                // a runtime environment object again.
400                $this->_runtimeEnvironment = null;
401            }
402        }
403
404        if (is_null($this->_runtimeEnvironment) || $this->_runtimeEnvironment->isCompatible()) {
405            return true;
406        }
407
408        return false;
