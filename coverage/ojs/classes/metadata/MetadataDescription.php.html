13 *
14 * @see MetadataProperty
15 * @see MetadataRecord
16 * @see MetadataSchema
17 *
18 * @brief Class modeling a description (DCMI abstract model) or subject-
19 *  predicate-object graph (RDF). This class and its children provide
20 *  meta-data (DCMI abstract model: statements of property-value pairs,
21 *  RDF: assertions of predicate-object pairs) about a given PKP application
22 *  entity instance (DCMI abstract model: described resource, RDF: subject).
23 *
24 *  This class has primarily been designed to describe journals, journal
25 *  issues, articles, conferences, conference proceedings (conference papers),
26 *  monographs (books), monograph components (book chapters) or citations.
27 *
28 *  It is, however, flexible enough to be extended to describe any
29 *  application entity in the future. Descriptions can be retrieved from
30 *  any application object that implements the MetadataProvider interface.
31 *
32 *  Special attention has been paid to the compatibility of the class
33 *  implementation with the implementation of several meta-data standards
34 *  that we consider especially relevant to our use cases.
35 *
36 *  We distinguish two main use cases for meta-data: discovery and delivery
37 *  of described resources. We have chosen the element-citation tag from the
38 *  NLM standard <http://dtd.nlm.nih.gov/publishing/tag-library/3.0/n-8xa0.html>
39 *  as our primary representation of delivery meta-data and dcterms
40 *  <http://dublincore.org/documents/dcmi-terms/> as our primary
41 *  representation of discovery meta-data.
42 *
43 *  Our specific use of meta-data has important implications and determines
44 *  our design goals:
45 *  * Neither NLM-citation nor dcterms have been designed with an object
46 *    oriented encoding in mind. NLM-citation is usually XML encoded
47 *    while typical dcterms encodings are HTML meta-tags, RDF or XML.
48 *  * We believe that trying to implement a super-set of meta-data
49 *    standards ("least common denominator" or super-schema approach)
50 *    is fundamentally flawed as meta-data standards are always
51 *    developed with specific use-cases in mind that require potentially
52 *    incompatible data properties or encodings.
53 *  * Although we think that NLM-citation and dcterms are sensible default
54 *    meta-data schemes our design should remain flexible enough for
55 *    users to implement and use other schemes as an internal meta-data
56 *    standard.
57 *  * We have to make sure that we can easily extract/inject meta-data
58 *    from/to PKP application objects.
59 *  * We have to avoid code duplication to keep maintenance cost under
60 *    control.
61 *  * We have to minimize the "impedance mismatch" between our own
62 *    object oriented encoding and fully standard compliant external
63 *    encodings (i.e. XML, RDF, HTML meta-tags, ...) to allow for easy
64 *    conversion between encodings.
65 *  * We have to make sure that we can switch between internal and
66 *    external encodings without any data loss.
67 *  * We have to make sure that crosswalks to and from other important
68 *    meta-data standards (e.g. OpenURL variants, MODS, MARC) can be
69 *    performed in a well-defined and easy way while minimizing data
70 *    loss.
71 *  * We have to make sure that we can support qualified fields (e.g.
72 *    qualified DC).
73 *  * We have to make sure that we can support RDF triples.
74 *
75 *  We took the following design decisions to achieve these goals:
76 *  * We only implement properties that are justified by strong real-world
77 *    use-cases. We recognize that the limiting factor is not the data that
78 *    we could represent but the data we actually have. This is not determined
79 *    by the chosen standard but by the PKP application objects we want to
80 *    represent. Additional meta-data properties/predicates can be added as
81 *    required.
82 *  * We do adapt data structures as long as we can make sure that a
83 *    fully standard compliant encoding can always be re-constructed. This
84 *    is especially true for NLM-citation which is designed with
85 *    XML in mind and therefore uses hierarchical constructs that are
86 *    difficult to represent in an OO class model.
87 *    This means that our meta-data framework only supports (nested) key/
88 *    value-based schemas which can however be converted to hierarchical
89 *    representations.
90 *  * We borrow class and property names from the DCMI abstract model as
91 *    the terms used there provide better readability for developers less
92 *    acquainted with formal model theory. We'll, however, make sure that
93 *    data can easily be RDF encoded within our data model.
94 *  * Data validation must ensure that meta-data always complies with a
95 *    specific meta-data standard. As we are speaking about an object
96 *    oriented encoding that is not defined in the original standard, we
97 *    define compliance as "roundtripability". This means we must be able
98 *    to convert our object oriented data encoding to a fully standard
99 *    compliant encoding and back without any data loss.
100 */

284    {

286        $property = & $this->getProperty($propertyName);
287        if (is_null($property)) {
288            return false;
289        }
290        assert($property instanceof \PKP\metadata\MetadataProperty);
291
292        // Check that the property is allowed for the described resource
293        if (!in_array($this->_assocType, $property->getAssocTypes())) {
294            return false;
295        }
296
297        // Handle translation
298        $translated = $property->getTranslated();
299        if (isset($locale) && !$translated) {
300            return false;
301        }
302        if (!isset($locale) && $translated) {
303            // Retrieve the current locale
304            $locale = Locale::getLocale();
305        }
306
307        // Check that the value is compliant with the property specification
308        if ($property->isValid($value, $locale) === false) {
309            return false;
310        }
311
312        // Handle cardinality
313        $existingValue = & $this->getStatement($propertyName, $locale);
314        switch ($property->getCardinality()) {
315            case MetadataProperty::METADATA_PROPERTY_CARDINALITY_ONE:
316                if (isset($existingValue) && !$replace) {
317                    return false;
318                }
319                $newValue = $value;
320                break;
321
322            case MetadataProperty::METADATA_PROPERTY_CARDINALITY_MANY:
323                if (isset($existingValue) && !$replace) {
324                    assert(is_array($existingValue));
325                    $newValue = $existingValue;
326                    array_push($newValue, $value);
327                } else {
328                    $newValue = [$value];
329                }
330                break;
331
332            default:
333                assert(false);
334        }
335
336        // Add the value
337        $this->setData($propertyName, $newValue, $locale);
338        return true;
339    }

342     * Remove statement. If the property has cardinality 'many'
343     * then all statements for the property will be removed at once.
344     * If the property is translated and the locale is null then
345     * the statements for all locales will be removed.

358        }

361    }

425    /**
426     * Add several statements at once. If one of the statements
427     * is invalid then the meta-data description will remain in its
428     * initial state.
429     * * Properties with a cardinality of 'many' must be passed in as
430     *   sub-arrays.
431     * * Translated properties with a cardinality of 'one' must be
432     *   passed in as sub-arrays with the locale as a key.
433     * * Translated properties with a cardinality of 'many' must be
434     *   passed in as sub-sub-arrays with the locale as the first key.
435     *
436     * @param array $statements statements
437     * @param int $replace one of the allowed replace levels.
438     *
439     * @return bool true if all statements could be added, false otherwise
440     */
441    public function setStatements(&$statements, $replace = self::METADATA_DESCRIPTION_REPLACE_PROPERTIES)
442    {
443        assert(in_array($replace, $this->_allowedReplaceLevels()));
444
445        // Make a backup copy of all existing statements.
446        $statementsBackup = $this->getAllData();
447
448        if ($replace == self::METADATA_DESCRIPTION_REPLACE_ALL) {
449            // Delete existing statements
450            $emptyArray = [];
451            $this->setAllData($emptyArray);
452        }
453
454        // Add statements one by one to detect invalid values.
455        foreach ($statements as $propertyName => $content) {
456            assert(!empty($content));
457
458            // Transform scalars or translated fields to arrays so that
459            // we can handle properties with different cardinalities in
460            // the same way.
461            if (is_scalar($content) || is_string(key($content))) {
462                $values = [&$content];
463            } else {
464                $values = & $content;
465            }
466
467            if ($replace == self::METADATA_DESCRIPTION_REPLACE_PROPERTIES) {
468                $replaceProperty = true;
469            } else {
470                $replaceProperty = false;
471            }
472
473            $valueIndex = 0;
474            foreach ($values as $value) {
475                $firstValue = ($valueIndex == 0) ? true : false;
476                // Is this a translated property?
477                if (is_array($value)) {
478                    foreach ($value as $locale => $translation) {
479                        // Handle cardinality many and one in the same way
480                        if (is_scalar($translation)) {
481                            $translationValues = [&$translation];
482                        } else {
483                            $translationValues = & $translation;
484                        }
485                        $translationIndex = 0;
486                        foreach ($translationValues as $translationValue) {
487                            $firstTranslation = ($translationIndex == 0) ? true : false;
488                            // Add a statement (replace existing statement if any)
489                            if (!($this->addStatement($propertyName, $translationValue, $locale, $firstTranslation && $replaceProperty))) {
490                                $this->setAllData($statementsBackup);
491                                return false;
492                            }
493                            unset($translationValue);
494                            $translationIndex++;
495                        }
496                        unset($translationValues);
497                    }
498                    unset($translation);
499                } else {
500                    // Add a statement (replace existing statement if any)
501                    if (!($this->addStatement($propertyName, $value, null, $firstValue && $replaceProperty))) {
502                        $this->setAllData($statementsBackup);
503                        return false;
504                    }
505                }
506                unset($value);
507                $valueIndex++;
508            }
509            unset($values);
510        }
511        return true;
512    }
