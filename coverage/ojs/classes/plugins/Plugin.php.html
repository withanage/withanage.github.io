22 * @brief Abstract class for plugins
23 *
24 * For best performance, a plug-in should not be instantiated if it is
25 * disabled or the current page/operation does not require the plug-in's
26 * functionality.
27 *
28 * Newer plug-ins support enable/disable and request filter settings that
29 * enable the PKP library plug-in framework to lazy-load plug-ins only

33 * do not support lazy-load because their register() method and hooks
34 * may have side-effects required on all requests. We have no way of
35 * knowing on which pages these side effects are important so we need
36 * to load legacy plug-ins on all pages.
37 *
38 * In these cases the register() function will be called on every request
39 * when the category the plug-in belongs to is being loaded. This was the
40 * default behavior before plug-in lazy load was introduced.
41 *
42 * Plug-ins that want to enable lazy-load have to include a 'lazy-load'
43 * setting in their version.xml:
44 *
